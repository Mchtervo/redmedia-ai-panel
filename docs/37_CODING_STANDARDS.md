# Coding Standards

# Redmedia AI Growth OS

Version 1.0

---

# Purpose

Define mandatory coding standards to ensure consistency, maintainability and scalability.

---

# General Principles

- Clean Code
- SOLID Principles
- DRY (Don't Repeat Yourself)
- KISS (Keep It Simple)
- Feature-First Architecture
- Strict TypeScript

---

# Naming Conventions

- PascalCase → Components
- camelCase → Variables & Functions
- kebab-case → Routes
- UPPER_SNAKE_CASE → Constants

---

# File Structure

- One responsibility per file
- Shared logic in reusable modules
- Co-locate tests with features

---

# Code Quality

- ESLint required
- Prettier formatting
- No unused imports
- No dead code
- Meaningful comments only

---

# Documentation

Every exported module must include:

- Purpose
- Inputs
- Outputs
- Example usage

---

# Success Criteria

Code should be readable, testable and production-ready.
