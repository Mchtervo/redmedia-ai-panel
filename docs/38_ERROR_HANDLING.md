# Error Flow

Detect

↓

Classify

↓

Log

↓

Notify

↓

Recover

↓

Learn

---

# Logging

Every error should include:

- Timestamp
- Company ID
- User ID
- Module
- Stack Trace
- Severity
- Correlation ID

---

# Recovery

- Retry
- Rollback
- Fallback
- Human Escalation

---

# User Experience

- Friendly error messages
- Actionable guidance
- No sensitive information exposed

---

# Success Criteria

Errors should be observable, recoverable and useful for continuous improvement.
